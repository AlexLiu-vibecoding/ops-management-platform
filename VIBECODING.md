> **⚠️ 每次开发前，AI 必须先读取此文件！**

## 零、Harness 协作模式（核心）

### 0.1 工作流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 1. 读取上下文 │ ──▶ │ 2. 确认需求   │ ──▶ │ 3. 按规格开发 │
│ VIBECODING   │     │ 填写规格模板  │     │ 对照检查清单  │
└──────────────┘     └──────────────┘     └──────────────┘
                                                │
                                                ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│ 6. 经验沉淀   │ ◀── │ 5. 更新文档   │ ◀── │ 4. 用户验收   │
│ 记录到本文档  │     │ CHANGELOG    │     │ 按模板确认   │
└──────────────┘     └──────────────┘     └──────────────┘
```

### 0.2 核心原则

| 原则 | 说明 |
|------|------|
| **先读后做** | 开发前必须先读取 VIBECODING.md |
| **先确认后开发** | 不确定的需求，先填规格模板确认 |
| **不擅自扩展** | 用户说什么做什么，不猜测不扩展 |
| **改完要验证** | 让用户验证效果，不自己认为完成 |
| **经验要沉淀** | 踩坑/学到的经验，记录到本文档 |

## 一、用户开发偏好

### 1.1 代码风格偏好

| 类型 | 偏好 |
|------|------|
| Python | Ruff 格式化，行宽 120 |
| 前端 | Vue 3 组合式 API，`<script setup>` |
| 表格操作列 | 统一使用 `TableActions` 组件 |
| 筛选区域 | 使用 `el-form inline` 布局 |

### 1.2 工作习惯

- **不擅自扩展范围**：用户说改什么就改什么
- **先确认再开发**：不确定的地方先问
- **改完要验证**：让用户确认效果

### 1.3 不喜欢的事情

| 行为 | 后果 |
|------|------|
| 用户只问问题，AI 却改了一堆代码 | ❌ 必须回滚 |
| 没确认需求就开始写代码 | ❌ 容易返工 |
| 代码写完不验证就说完成 | ❌ 不信任 |
| 重复问已经确认过的问题 | ❌ 浪费 token |

## 二、容易遗漏的事项

| 场景 | 容易遗漏 | 正确做法 | 检查项 |
|------|----------|----------|--------|
| 新增 API | 忘记添加测试用例 | 同步更新 `tests/test_xxx_api.py` | □ |
| 新增菜单 | 忘记更新数据库 | 提示调用 `/api/v1/menu/add-missing` | □ |
| 修改前端 | 忘记确认页面效果 | 让用户刷新验证 | □ |
| 新增字段 | 忘记更新 Schema | 同步更新 Pydantic 模型 | □ |
| 开发完成 | 忘记更新文档 | 更新 CHANGELOG.md | □ |
| 踩坑/学新东西 | 忘记记录经验 | 更新本文档 | □ |

## 三、项目特定经验

### 3.1 菜单/路由修改

**新增菜单需要同时修改：**
1. `backend/app/api/menu.py` - 菜单数据
2. `frontend/src/router/index.js` - 路由配置
3. 数据库 - 调用 `/api/v1/menu/add-missing`

**经验教训：**
- `add-missing` API 会删除 `old_paths` 列表中的旧菜单
- 移动菜单时小心别误删

### 3.2 测试用例

**位置：** `backend/tests/`
**命名：** `test_xxx_api.py` / `test_xxx_flow.py`
**习惯：** 新增 API 后，用户会提醒添加测试用例

### 3.3 日志位置

```bash
/app/work/logs/bypass/app.log      # 后端主日志
/app/work/logs/bypass/console.log  # 前端控制台
/app/work/logs/bypass/dev.log      # 开发环境
```

## 四、踩坑记录

### 4.1 端口检测

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| `lsof -i:5000` | `curl -I http://localhost:5000` |
| 原因：会检测到 50001 等 | 或 `ss -tuln \| grep -E ':5000[[:space:]]'` |

### 4.2 文件下载

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| `<a href=url download>` | `fetch + blob + createObjectURL` |
| 原因：跨域 URL download 属性被忽略 | |

### 4.3 热更新机制

- **前端**：Vite 热更新，修改后无需重启
- **后端**：FastAPI + uvicorn 默认热重载
- **做法**：改完代码让用户刷新验证

### 4.4 需求理解

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| 用户问"飞书配置是自带的吗？" | 只回答：是的，已集成 |
| AI 回答后又改了一堆代码 | 不要擅自改代码 |

```markdown
   - [ ] 功能点2

3. 请验证：
   - 刷新页面查看 xxx 功能
   - 测试 xxx 场景
```
