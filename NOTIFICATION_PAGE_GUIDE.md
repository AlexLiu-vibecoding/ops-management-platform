## 功能演示

### 添加静默规则

点击"添加规则"按钮后，弹出对话框：

```
┌─────────────────────────────────────────┐
│         添加静默规则                    │
├─────────────────────────────────────────┤
│ 规则名称：[周末维护窗口静默          ] │

│                                         │
│ 时间段：  [02:00] - [06:00]           │
│                                         │
│ 生效星期：□周一 □周二 □周三 □周四     │
│          □周五 ☑周六 ☑周日            │
│                                         │
│ 匹配条件：                              │
│ ┌─────────────┬──────────────┐        │
│ │ 指标类型 ▼ │ 告警级别 ▼  │        │
│ └─────────────┴──────────────┘        │
│                                         │
│ 优先级：[10]                            │
│                                         │
│ 是否启用：[√]                           │
│                                         │
│ 描述：[周末维护时间段静默所有告警      ]│
│                                         │
│           [取消]  [保存]               │
└─────────────────────────────────────────┘
```

### 静默类型说明

1. **一次性静默**
   - 在指定日期范围内生效
   - 适用于临时维护、系统升级等场景
   - 示例：2024-04-02 至 2024-04-03

2. **每日重复**
   - 每天在指定时间段内生效
   - 适用于日常维护窗口
   - 示例：每天 02:00 - 06:00

3. **每周重复**
   - 每周在指定星期和时间段生效
   - 适用于周末维护、定期巡检等场景
   - 示例：周六、周日 00:00 - 23:59

### 添加频率限制规则

```
┌─────────────────────────────────────────┐
│         添加频率限制规则                │
├─────────────────────────────────────────┤
│ 规则名称：[告警频率限制              ] │
│                                         │
│ 时间窗口(秒)：[300]                     │
│                                         │
│ 最大通知数：[5]                         │
│                                         │
│ 冷却期(秒)：[600]                       │
│                                         │
│ 匹配条件：                              │
│ ┌─────────────┬──────────────┐        │
│ │ 指标类型 ▼ │ 告警级别 ▼  │        │
│ │ CPU使用率  │ 严重         │        │
│ └─────────────┴──────────────┘        │
│                                         │
│ 优先级：[10]                            │
│                                         │
│ 是否启用：[√]                           │
│                                         │
│ 描述：[限制CPU告警的发送频率          ]│
│                                         │
│           [取消]  [保存]               │
└─────────────────────────────────────────┘
```

### 频率限制说明

- **时间窗口**：统计通知数量的时间范围（如300秒=5分钟）
- **最大通知数**：时间窗口内最多发送的通知次数
- **冷却期**：超过限制后，需等待的时间才能继续发送

**示例**：
- 时间窗口：300秒
- 最大通知数：5次
- 冷却期：600秒

**效果**：
- 5分钟内最多发送5次通知
- 超过5次后，需要等待10分钟才能继续发送

## 权限控制

### 按钮权限

根据用户权限，按钮显示如下：

| 角色 | 查看规则 | 添加规则 | 编辑规则 | 删除规则 |
|-----|---------|---------|---------|---------|
| super_admin | ✅ | ✅ | ✅ | ✅ |
| approval_admin | ✅ | ❌ | ❌ | ❌ |
| operator | ✅ | ❌ | ❌ | ❌ |

### 权限验证

- 所有添加/编辑/删除操作都需要对应权限
- 无权限时，按钮不显示或禁用
- API层也会进行权限验证

## 使用场景

### 场景1：周末维护窗口静默

```
规则名称：周末维护窗口静默
静默类型：每周重复
时间段：02:00 - 06:00
生效星期：周六、周日
优先级：10
```

**效果**：每周六、周日的02:00-06:00期间，不发送任何告警通知

### 场景2：系统升级临时静默

```
规则名称：系统升级静默
静默类型：一次性
生效日期：2024-04-02 至 2024-04-03
优先级：20
```

**效果**：在2024-04-02至2024-04-03期间，不发送任何告警通知

### 场景3：CPU告警频率限制

```
规则名称：CPU告警频率限制
指标类型：CPU使用率
时间窗口：300秒
最大通知数：3次
冷却期：600秒
```

**效果**：CPU告警在5分钟内最多发送3次，超过后需等待10分钟

## 操作步骤

### 1. 创建静默规则

1. 点击"静默规则"区域的"添加规则"按钮
2. 填写规则名称（必填）
3. 选择静默类型（一次性/每日/每周）
4. 配置时间段或日期范围
5. （可选）配置匹配条件
6. 设置优先级和启用状态
7. 点击"保存"

### 2. 创建频率限制规则

1. 点击"频率限制"区域的"添加规则"按钮
2. 填写规则名称（必填）
3. 配置时间窗口、最大通知数、冷却期
4. （可选）配置匹配条件
5. 设置优先级和启用状态
6. 点击"保存"

### 3. 编辑规则

1. 在规则列表中找到目标规则
2. 点击"编辑"按钮
3. 修改配置
4. 点击"保存"

### 4. 删除规则

1. 在规则列表中找到目标规则
2. 点击"删除"按钮
3. 确认删除操作

## 注意事项

1. **优先级**：数值越大，优先级越高，优先匹配
2. **时间段**：使用24小时制，格式为HH:MM
3. **星期**：周一=0，周二=1，...，周日=6
4. **生效顺序**：静默规则 → 频率限制 → 通知发送
5. **权限要求**：需要相应的权限才能进行操作

## 故障排查

### 问题1：看不到"添加规则"按钮

**原因**：当前用户没有相应权限

**解决**：联系管理员分配权限
- 静默规则：需要 `notification:silence_manage` 权限
- 频率限制：需要 `notification:rate_limit_manage` 权限

### 问题2：保存失败

**可能原因**：
1. 规则名称已存在
2. 必填字段未填写
3. 时间格式错误

**解决**：检查输入数据，确保格式正确

### 问题3：规则不生效

**可能原因**：
1. 规则未启用
2. 优先级过低，被其他规则覆盖
3. 匹配条件不满足

**解决**：检查规则配置，调整优先级
