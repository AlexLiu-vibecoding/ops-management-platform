# VibeCoding 协作文档

> AI 与用户协作开发的知识库，提高沟通成功率，减少返工。

---

## 文档结构

```
.vibecoding/
├── README.md           # 本文件，快速导航
├── VIBECODING.md       # ⭐ 协作经验（每次开发前必读）
├── HARNESS.md          # 协作框架总览
├── CHECKLIST.md        # 开发检查清单
├── CHANGELOG.md        # 变更记录
├── DEVELOPMENT.md      # 技术规范
└── specs/
    └── template.md     # 功能规格模板
```

---

## 快速使用

### AI 侧

```
1. 每次开发前：读取 .vibecoding/VIBECODING.md
2. 复杂需求：填写 specs/template.md
3. 开发过程中：对照 CHECKLIST.md
4. 开发完成后：按验收模板确认
5. 有新经验：更新 VIBECODING.md
```

### 用户侧

```
- 开发开始时说："先看 VIBECODING"
- 新功能说："请先写规格"
- 踩坑/新偏好说："记录到 VIBECODING"
```

## 文档职责

| 文档 | 职责 | 何时使用 |
|------|------|----------|
| **VIBECODING.md** | 协作经验、用户偏好、踩坑记录 | ⭐ 每次开发前必读 |
| HARNESS.md | 协作框架总览 | 了解整体协作模式 |
| CHECKLIST.md | 开发检查清单 | 开发过程中对照 |
| CHANGELOG.md | 变更记录 | 每次变更后记录 |

| specs/template.md | 功能规格模板 | 新功能开发前填写 |
