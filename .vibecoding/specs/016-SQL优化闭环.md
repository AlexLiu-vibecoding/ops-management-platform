# SQL 性能优化闭环

> 自动采集慢SQL → 周期性分析 → 优化建议 → 一键提交变更 → 效果验证
> 
> **新增**：支持慢日志文件上传分析，文件保留30天，分析历史保留1年

---

## 基本信息

- **规格编号**: `SPEC-016`
- **功能名称**: SQL 性能优化闭环
- **优先级**: P1 (重要功能)
- **状态**: 开发中

## 一、需求背景

### 1.1 问题描述

当前 SQL 优化流程存在断层：
- 慢查询监控和 SQL 优化器是独立模块
- 发现慢 SQL 后需要人工复制到优化器分析
- 优化后需要手动创建变更申请
- 缺乏优化效果追踪
- **新增**：无法分析用户上传的慢日志文件

### 1.2 目标

实现 SQL 性能优化的完整闭环：
1. 自动采集慢 SQL（定时任务）
2. **新增**：支持上传慢日志文件分析
3. 周期性 LLM 分析，生成优化建议
4. 一键采用建议，自动提交变更申请
5. 变更执行后自动验证效果

### 1.3 数据保留策略

| 数据类型 | 保留时间 | 清理方式 |
|---------|---------|---------|
| 上传的慢日志文件 | 30天 | 定时任务自动清理（每天凌晨3点） |
| 分析历史记录 | 1年 | 定时任务自动清理（每天凌晨4点） |

### 1.4 不做的事

- 不自动执行变更（必须走审批流程）
- 不采集生产数据的实际内容
- 不替代 DBA 的最终决策

## 三、数据模型（精确约束）

### 3.1 新增表：优化建议表

```sql
CREATE TABLE sql_optimization_suggestions (
    id SERIAL PRIMARY KEY,
    instance_id INTEGER NOT NULL REFERENCES rdb_instances(id),
    database_name VARCHAR(100),
    
    -- 慢SQL信息
    slow_query_id INTEGER REFERENCES slow_queries(id),
    sql_fingerprint TEXT NOT NULL,
    sql_sample TEXT,
    
    -- 分析结果
    issues JSONB,                    -- 问题列表
    suggestions JSONB NOT NULL,      -- 优化建议
    suggested_sql TEXT,              -- 建议的SQL（改写后）
    index_ddl TEXT,                  -- 建议的索引DDL
    rollback_sql TEXT,               -- 回滚SQL（如DROP INDEX）
    
    -- 风险评估
    risk_level VARCHAR(20) DEFAULT 'low',  -- low/medium/high
    confidence DECIMAL(3,2),         -- LLM置信度 0-1
    expected_improvement VARCHAR(100), -- 预期提升
    
    -- 状态管理
    status VARCHAR(20) DEFAULT 'pending',  -- pending/adopted/rejected/expired
    approval_id INTEGER,             -- 关联的变更申请ID
    adopted_by INTEGER,              -- 采用人
    adopted_at TIMESTAMP,
    
    -- 效果验证
    before_avg_time DECIMAL(10,3),   -- 优化前平均执行时间
    after_avg_time DECIMAL(10,3),    -- 优化后平均执行时间
    actual_improvement DECIMAL(5,2), -- 实际提升百分比
    
    -- 分析元数据
    analysis_time DECIMAL(5,2),      -- 分析耗时（秒）
    llm_model VARCHAR(50),           -- 使用的模型
    table_schemas_used JSONB,        -- 分析时使用的表结构
    
    -- 时间戳
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- 索引
    UNIQUE(instance_id, sql_fingerprint, created_at)
);

CREATE INDEX idx_suggestions_status ON sql_optimization_suggestions(status);
CREATE INDEX idx_suggestions_instance ON sql_optimization_suggestions(instance_id);
CREATE INDEX idx_suggestions_created ON sql_optimization_suggestions(created_at DESC);
```

CREATE TABLE slow_query_collection_tasks (
    id SERIAL PRIMARY KEY,
    instance_id INTEGER NOT NULL REFERENCES rdb_instances(id),
    
    -- 任务配置
    enabled BOOLEAN DEFAULT TRUE,
    cron_expression VARCHAR(50) DEFAULT '0 */5 * * *',  -- 默认每5分钟
    min_exec_time DECIMAL(5,2) DEFAULT 1.0,             -- 最小执行时间阈值
    top_n INTEGER DEFAULT 100,                          -- 每次采集Top N
    
    -- 自动分析配置
    auto_analyze BOOLEAN DEFAULT TRUE,                  -- 是否自动分析
    analyze_threshold INTEGER DEFAULT 3,                -- 执行次数超过阈值才分析
    llm_analysis BOOLEAN DEFAULT TRUE,                  -- 是否启用LLM分析
    
    -- 统计信息
    last_run_at TIMESTAMP,
    last_run_status VARCHAR(20),
    last_collected_count INTEGER DEFAULT 0,
    total_collected_count INTEGER DEFAULT 0,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_collection_tasks_instance ON slow_query_collection_tasks(instance_id);
CREATE INDEX idx_collection_tasks_enabled ON slow_query_collection_tasks(enabled);

## 七、核心流程

### 7.1 自动采集流程

```
定时任务触发
    │
    ▼
获取启用的采集任务列表
    │
    ▼
遍历每个任务
    │
    ├── 连接实例，查询慢SQL
    │
    ├── 去重（与现有记录对比）
    │
    ├── 存储到 SlowQuery 表
    │
    └── 如果 auto_analyze=True 且执行次数 >= threshold
        │
        └── 调用分析服务，生成优化建议
```

### 7.2 采用建议流程

```
用户点击"采用"
    │
    ▼
检查建议状态（必须是pending）
    │
    ▼
创建变更申请
    │
    ├── 变更类型：DDL（索引）或 SQL改写
    ├── 变更SQL：index_ddl 或 suggested_sql
    ├── 回滚SQL：rollback_sql
    ├── 风险等级：risk_level
    └── 关联优化建议ID
    │
    ▼
更新建议状态为 adopted
    │
    ▼
返回变更申请ID供用户查看
```

### 7.3 文件上传分析流程（新增）

```
用户上传慢日志文件
    │
    ▼
验证文件格式（.log/.txt）
    │
    ▼
计算文件哈希，检查是否重复
    │
    ├── 重复：返回已有记录
    │
    └── 新文件：
        │
        ├── 保存到 /tmp/slow_log_files/
        │
        ├── 创建 SlowLogFile 记录
        │
        ├── 设置过期时间（30天后）
        │
        └── 如果 auto_analyze=True
            │
            ├── 解析慢日志（MySQL格式）
            │
            ├── 对每个SQL进行分析
            │   │
            │   ├── 生成SQL指纹
            │   │
            │   ├── 规则检测（查询时间、扫描比等）
            │   │
            │   └── 创建 SQLAnalysisHistory 记录
            │
            └── 返回分析结果
```

### 7.4 数据清理流程（新增）

```
定时任务（每天凌晨执行）
    │
    ├── 3:00 清理过期慢日志文件
    │   │
    │   ├── 查找 expire_at <= now 的文件
    │   │
    │   ├── 删除物理文件
    │   │
    │   ├── 删除关联的分析历史
    │   │
    │   └── 删除数据库记录
    │
    └── 4:00 清理过期分析历史
        │
        ├── 查找 expire_at <= now 的记录
        │
        └── 删除数据库记录
```
