## 三、核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/menu/list` | GET | 菜单列表 |
| `/api/v1/menu` | POST | 创建菜单 |
| `/api/v1/menu/{id}` | PUT | 更新菜单 |
| `/api/v1/menu/{id}` | DELETE | 删除菜单 |
| `/api/v1/menu/add-missing` | POST | 添加缺失菜单 |
| `/api/v1/system/config` | GET | 系统配置 |
| `/api/v1/system/config` | PUT | 更新配置 |

---

## 四、菜单模型

```python
class MenuConfig:
    id: int
    parent_id: int         # 父菜单ID
    name: str              # 菜单名称
    path: str              # 路由路径
    icon: str              # 图标
    component: str         # 组件路径
    sort_order: int        # 排序
    is_visible: bool       # 是否显示
    is_enabled: bool       # 是否启用
    roles: str             # 允许的角色（逗号分隔）
```

---

## 五、注意事项

- 菜单修改需要调用 `/menu/add-missing` 更新数据库
- 存储策略修改需要测试可用性
- 部分配置需要重启服务生效
