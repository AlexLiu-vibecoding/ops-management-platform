# 巡检报告

> 定时巡检、巡检报告生成

---

## 基本信息

- **规格编号**: `SPEC-013`
- **功能名称**: 巡检报告
- **优先级**: P2 (一般功能)
- **状态**: ✅ 已完成

---

## 一、功能概述

### 1.1 巡检项

| 项目 | 说明 |
|------|------|
| 连接数 | 当前连接数、最大连接数 |
| 慢查询 | 慢查询数量、TOP 慢查询 |
| 锁状态 | 锁等待、死锁 |
| 主从延迟 | 复制延迟 |
| 磁盘空间 | 表空间、磁盘使用 |
| 索引效率 | 无效索引、重复索引 |

### 1.2 核心功能

- 手动巡检
- 定时巡检
- 巡检报告查看
- 巡检历史记录

---

## 二、实现位置

### 2.1 后端

| 文件 | 说明 |
|------|------|
| `backend/app/api/inspection.py` | 巡检 API |
| `backend/app/api/scheduled_inspection.py` | 定时巡检 API |

### 2.2 前端

| 文件 | 说明 |
|------|------|
| `frontend/src/views/inspection/index.vue` | 巡检报告 |
| `frontend/src/views/inspection/scheduled/index.vue` | 定时巡检 |

---

## 三、核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/inspection/run` | POST | 执行巡检 |
| `/api/v1/inspection/reports` | GET | 巡检报告列表 |
| `/api/v1/inspection/reports/{id}` | GET | 报告详情 |
| `/api/v1/scheduled-inspection` | GET | 定时巡检列表 |
| `/api/v1/scheduled-inspection` | POST | 创建定时巡检 |

---

## 四、巡检流程

```
1. 选择实例
2. 执行巡检项检查
3. 汇总结果
4. 生成报告
5. 发送通知（如配置）
```

---

## 五、注意事项

- 巡检会消耗数据库资源
- 生产环境建议在低峰期执行
- 定时巡检可配置通知
