# 环境管理

> 开发/测试/预发/生产环境隔离，环境权限控制

---

## 基本信息

- **规格编号**: `SPEC-003`
- **功能名称**: 环境管理
- **优先级**: P0 (核心功能)
- **状态**: ✅ 已完成

---

## 一、功能概述

### 1.1 环境类型

| 环境 | 说明 |
|------|------|
| dev | 开发环境 |
| test | 测试环境 |
| staging | 预发环境 |
| prod | 生产环境 |

### 1.2 核心功能

- 环境 CRUD 管理
- 环境权限绑定（基于角色）
- 环境内实例管理
- AWS 凭证配置（RDS 采集用）

## 三、核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/environments` | GET | 环境列表 |
| `/api/v1/environments` | POST | 创建环境 |
| `/api/v1/environments/{id}` | PUT | 更新环境 |
| `/api/v1/environments/{id}` | DELETE | 删除环境 |
| `/api/v1/environments/{id}/permissions` | GET | 获取环境权限 |
| `/api/v1/environments/{id}/permissions` | POST | 设置环境权限 |

---

## 四、权限模型

环境权限从绑定用户改为绑定角色：

```python
class EnvironmentPermission:
    environment_id: int
    role: str          # super_admin/approval_admin/operator/developer
    permissions: list  # ['view', 'execute', 'admin']
```

---

## 五、注意事项

- 生产环境需要更高权限才能操作
- AWS 凭证用于 RDS CloudWatch 采集
- 环境删除前需要检查关联实例
