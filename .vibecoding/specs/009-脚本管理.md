# 脚本管理

> Python/Bash/SQL 脚本管理、批量执行

---

## 基本信息

- **规格编号**: `SPEC-009`
- **功能名称**: 脚本管理
- **优先级**: P1 (重要功能)
- **状态**: ✅ 已完成

---

## 一、功能概述

### 1.1 支持的脚本类型

| 类型 | 说明 |
|------|------|
| Python | Python 脚本 |
| Bash | Shell 脚本 |
| SQL | SQL 脚本 |

### 1.2 核心功能

- 脚本 CRUD 管理
- 脚本执行
- 批量执行（多实例）
- 执行历史记录
- 执行结果查看

---

## 二、实现位置

### 2.1 后端

| 文件 | 说明 |
|------|------|
| `backend/app/api/scripts.py` | 脚本 API |
| `backend/app/models/script.py` | 脚本模型 |

### 2.2 前端

| 文件 | 说明 |
|------|------|
| `frontend/src/views/scripts/index.vue` | 脚本管理页面 |

---

## 三、核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/scripts` | GET | 脚本列表 |
| `/api/v1/scripts` | POST | 创建脚本 |
| `/api/v1/scripts/{id}` | GET | 脚本详情 |
| `/api/v1/scripts/{id}` | PUT | 更新脚本 |
| `/api/v1/scripts/{id}` | DELETE | 删除脚本 |
| `/api/v1/scripts/{id}/execute` | POST | 执行脚本 |
| `/api/v1/scripts/{id}/history` | GET | 执行历史 |

---

## 四、执行权限

| 角色 | 权限 |
|------|------|
| super_admin | 全部脚本 |
| approval_admin | 全部脚本 |
| operator | 指定脚本 |
| developer | 无执行权限 |

---

## 五、注意事项

- 脚本执行在沙箱环境中
- 危险操作需要审批
- 执行结果保存到日志
