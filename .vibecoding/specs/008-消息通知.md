## 四、通知绑定

将通知类型与通道关联：

```
通知类型 + 通道 = 自动推送
```

例如：
- 告警通知 → 钉钉运维群
- 审批通知 → 企业微信审批群

## 五、通知配置功能 (新增)

### 5.1 静默规则

在指定时间范围内静默特定类型的通知，适用于：
- 维护窗口期
- 计划内的系统停机
- 避免夜间打扰

**规则字段**：
```json
{
  "name": "维护窗口静默",
  "notification_type": "alert",
  "start_time": "2024-01-01T22:00:00",
  "end_time": "2024-01-02T06:00:00",
  "enabled": true
}
```

### 5.2 频率限制

控制通知发送频率，避免通知轰炸：

**限制字段**：
```json
{
  "name": "告警频率限制",
  "notification_type": "alert",
  "max_count": 10,
  "time_window": 3600,  // 秒
  "enabled": true
}
```

**实现策略**：
- 采用内存缓存记录发送次数
- 超过限制的通知将被丢弃
- 时间窗口结束后自动重置计数

## 八、开发经验

### 8.1 踩坑记录

#### 问题：导航栏菜单不显示
- **原因**：只配置了前端路由，未配置数据库菜单
- **解决**：在 `menu_configs` 表中添加菜单记录
- **教训**：本项目采用数据库驱动菜单，前端路由和数据库菜单需同步配置

#### 问题：API 路径重复导致 404
- **原因**：前端调用时手动加了 `/api/v1` 前缀，而 baseURL 已包含
- **解决**：移除前端代码中的重复前缀
- **教训**：使用 API 前先了解 baseURL 配置

### 8.2 最佳实践

1. **开发前必读 AGENTS.md** - 了解项目架构和开发规范
2. **完整验证** - 功能开发完成后验证全流程（菜单显示、权限控制等）
3. **错误处理** - 解密等敏感操作需添加异常处理
4. **代码规范** - 遵循项目既定的代码风格
