# 消息通知

> 钉钉/企业微信/飞书/邮件/Webhook 通知

---

## 基本信息

- **规格编号**: `SPEC-008`
- **功能名称**: 消息通知
- **优先级**: P1 (重要功能)
- **状态**: ✅ 已完成

---

## 一、功能概述

### 1.1 支持的通道

| 通道 | 说明 |
|------|------|
| 钉钉 | 机器人 Webhook |
| 企业微信 | 机器人 Webhook |
| 飞书 | 机器人 Webhook |
| 邮件 | SMTP 发送 |
| 自定义 Webhook | 通用 HTTP 回调 |

### 1.2 验证方式

| 方式 | 说明 |
|------|------|
| 无验证 | 直接发送 |
| 关键词 | 消息包含关键词 |
| 签名 | 加签验证 |
| Token | Token 验证 |

### 1.3 通知类型

| 类型 | 说明 |
|------|------|
| 审批通知 | 变更审批相关 |
| 告警通知 | 监控告警 |
| 定时任务通知 | 任务执行结果 |
| 审计日志通知 | 重要操作通知 |

## 三、核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/notification/channels` | GET | 通道列表 |
| `/api/v1/notification/channels` | POST | 创建通道 |
| `/api/v1/notification/channels/{id}` | PUT | 更新通道 |
| `/api/v1/notification/channels/{id}` | DELETE | 删除通道 |
| `/api/v1/notification/channels/{id}/test` | POST | 测试通道 |
| `/api/v1/notification/bindings` | GET | 绑定列表 |
| `/api/v1/notification/bindings` | POST | 创建绑定 |

---

## 四、通知绑定

将通知类型与通道关联：

```
通知类型 + 通道 = 自动推送
```

例如：
- 告警通知 → 钉钉运维群
- 审批通知 → 企业微信审批群

---

## 五、注意事项

- Webhook 地址保密存储
- 测试功能可验证配置正确性
- 邮件需要配置 SMTP 服务器
