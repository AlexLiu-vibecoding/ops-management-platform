# 变更审批

> DDL/DML 变更审批、风险分析、回滚SQL生成、分库分表支持

---

## 基本信息

- **规格编号**: `SPEC-005`
- **功能名称**: 变更审批
- **优先级**: P0 (核心功能)
- **状态**: ✅ 已完成

---

## 一、功能概述

### 1.1 变更流程

```
提交申请 → 风险分析 → 审批 → 执行 → 验证
```

### 1.2 支持的变更类型

| 类型 | 说明 |
|------|------|
| MySQL DDL | CREATE/ALTER/DROP |
| MySQL DML | INSERT/UPDATE/DELETE |
| Redis 命令 | SET/DEL/EXPIRE 等 |

### 1.3 核心功能

- 变更申请提交
- 自动风险分析
- 审批流程
- 自动生成回滚 SQL
- 分库分表支持
- 变更时间窗口
- 定时执行

## 四、回滚 SQL 生成

### MySQL DDL
- CREATE TABLE → DROP TABLE
- ALTER TABLE → 反向 ALTER
- DROP TABLE → 无法生成（需备份）

### MySQL DML
- INSERT → DELETE
- UPDATE → 反向 UPDATE
- DELETE → INSERT（需先查询数据）

### Redis
- SET → DEL
- DEL → 无法生成（需先备份）
- EXPIRE → PERSIST 或恢复原 TTL

## 五、分库分表支持

| 模式 | 示例 |
|------|------|
| 单库选择 | `db1.table` |
| 多库选择 | 选择多个数据库 |
| 通配符匹配 | `db_%.table` 或 `user_db_*.table` |
| 全部数据库 | 所有匹配的库执行 |

---

## 六、注意事项

- 高风险变更需要多人审批
- 生产环境变更需要在时间窗口内执行
- 执行前自动生成回滚 SQL
- 大 SQL 文件自动切换到文件存储
