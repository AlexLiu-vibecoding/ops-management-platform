## 一、需求背景

### 1.1 问题描述

当前系统 AI 分析功能仅支持单一模型调用，存在以下问题：
1. **稳定性不足**：单点故障，模型服务不可用时 AI 分析完全失效
2. **无法比对验证**：无法使用多模型交叉验证分析结果
3. **私有化受限**：私有化部署用户希望调用本地模型（如 Ollama、vLLM），当前架构不支持
4. **配置不灵活**：API 密钥、调用地址等硬编码或依赖环境变量，无法在界面配置

### 1.2 目标

1. **多模型支持**：配置多个 AI 模型通道，支持不同提供商（豆包、OpenAI、本地模型等）
2. **高可用性**：支持主备切换，主模型不可用时自动切换到备用模型
3. **交叉验证**：支持同时调用多个模型，对比分析结果
4. **私有化友好**：支持配置本地模型地址，适配私有化部署场景
5. **界面配置**：前端提供配置界面，支持配置 API 地址、密钥、模型参数等

### 1.3 不做的事（边界）

1. 不实现模型训练、微调功能
2. 不实现模型代理网关（仅作为客户端调用）
3. 不支持模型计费、配额管理
4. 不实现流式响应的前端展示（本次仅后端使用）

## 二、数据模型

### 2.1 数据库表结构

```sql
-- AI 模型配置表
CREATE TABLE ai_model_configs (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,              -- 配置名称
    provider VARCHAR(50) NOT NULL,                   -- 提供商: openai/doubao/ollama/custom
    base_url VARCHAR(500) NOT NULL,                  -- API 地址
    api_key_encrypted VARCHAR(500),                  -- 加密后的 API 密钥
    model_name VARCHAR(100) NOT NULL,                -- 模型标识
    model_type VARCHAR(30) DEFAULT 'chat',           -- 模型类型: chat/completion/embedding
    
    -- 调用参数
    max_tokens INTEGER DEFAULT 4096,                 -- 最大 token 数
    temperature REAL DEFAULT 0.7,                    -- 温度参数
    timeout INTEGER DEFAULT 30,                      -- 超时时间(秒)
    
    -- 状态与优先级
    is_enabled BOOLEAN DEFAULT TRUE,                 -- 是否启用
    is_default BOOLEAN DEFAULT FALSE,                -- 是否默认通道
    priority INTEGER DEFAULT 0,                      -- 优先级(越大越高)
    
    -- 使用场景
    use_cases JSONB DEFAULT '[]',                    -- 使用场景: ["alert_analysis", "sql_optimize"]
    
    -- 元数据
    description VARCHAR(200),                        -- 描述
    created_by INTEGER REFERENCES users(id),         -- 创建人
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 创建时间
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP   -- 更新时间
);

-- AI 调用日志表
CREATE TABLE ai_call_logs (
    id SERIAL PRIMARY KEY,
    model_config_id INTEGER REFERENCES ai_model_configs(id),
    use_case VARCHAR(50) NOT NULL,                   -- 使用场景
    input_tokens INTEGER,                            -- 输入 token 数
    output_tokens INTEGER,                           -- 输出 token 数
    latency_ms INTEGER,                              -- 响应耗时(毫秒)
    success BOOLEAN NOT NULL,                        -- 是否成功
    error_message TEXT,                              -- 错误信息
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 创建索引
CREATE INDEX idx_ai_model_configs_enabled ON ai_model_configs(is_enabled);
CREATE INDEX idx_ai_model_configs_priority ON ai_model_configs(priority DESC);
CREATE INDEX idx_ai_call_logs_model ON ai_call_logs(model_config_id);
CREATE INDEX idx_ai_call_logs_time ON ai_call_logs(created_at);
```

## 四、业务逻辑

### 4.1 模型选择策略

```python
def get_available_model(use_case: str) -> Optional[AIModelConfig]:
    """
    获取可用的模型配置
    
    优先级策略:
    1. 按优先级降序排列
    2. 只选择启用且匹配使用场景的模型
    3. 如果有默认模型且可用，优先使用
    """
    models = db.query(AIModelConfig).filter(
        AIModelConfig.is_enabled == True,
        AIModelConfig.use_cases.contains([use_case])
    ).order_by(AIModelConfig.priority.desc()).all()
    
    # 优先使用默认模型
    for model in models:
        if model.is_default:
            return model
    
    return models[0] if models else None
```

### 4.2 多模型调用策略

```python
async def call_with_fallback(
    use_case: str,
    prompt: str,
    fallback: bool = True
) -> str:
    """
    带降级的模型调用
    
    Args:
        use_case: 使用场景
        prompt: 提示词
        fallback: 是否启用降级（主模型失败时切换备用）
    """
    models = get_models_by_priority(use_case)
    
    errors = []
    for model in models:
        try:
            result = await call_model(model, prompt)
            log_call(model.id, success=True)
            return result
        except Exception as e:
            errors.append((model.name, str(e)))
            log_call(model.id, success=False, error=str(e))
            if not fallback:
                raise
    
    raise AIModelUnavailableError(f"所有模型都不可用: {errors}")
```

### 4.3 交叉验证调用

```python
async def call_with_cross_validation(
    use_case: str,
    prompt: str,
    model_count: int = 2
) -> CrossValidationResult:
    """
    多模型交叉验证调用
    
    同时调用多个模型，返回所有结果供对比
    """
    models = get_models_by_priority(use_case)[:model_count]
    
    tasks = [call_model(model, prompt) for model in models]
    results = await asyncio.gather(*tasks, return_exceptions=True)
    
    return CrossValidationResult(
        models=[m.name for m in models],
        responses=results,
        consensus=analyze_consensus(results)  # 分析一致性
    )
```

### 4.4 API 密钥加密存储

# 使用 AES 加密存储 API 密钥
from app.utils.encryption import aes_cipher

def save_api_key(config: AIModelConfig, api_key: str):
    config.api_key_encrypted = aes_cipher.encrypt(api_key)

def get_api_key(config: AIModelConfig) -> str:
    if not config.api_key_encrypted:
        return ""
    return aes_cipher.decrypt(config.api_key_encrypted)

## 七、安全考虑

1. **密钥加密存储**：API 密钥使用 AES 加密后存储
2. **密钥脱敏展示**：前端只显示密钥后4位，如 `sk-****xxxx`
3. **权限控制**：只有 super_admin 可以管理模型配置
4. **日志审计**：所有模型调用记录日志，包含 token 消耗和耗时
5. **调用限流**：可配置调用频率限制，防止滥用
