# 定时任务

> 定时执行脚本、审批定时执行

---

## 基本信息

- **规格编号**: `SPEC-010`
- **功能名称**: 定时任务
- **优先级**: P1 (重要功能)
- **状态**: ✅ 已完成

---

## 一、功能概述

### 1.1 核心功能

- 定时任务 CRUD
- Cron 表达式调度
- 任务执行历史
- 任务状态管理（启用/禁用/暂停）
- 定时审批执行

### 1.2 调度器

使用 APScheduler 实现定时调度。

## 三、核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/scheduled-tasks` | GET | 任务列表 |
| `/api/v1/scheduled-tasks` | POST | 创建任务 |
| `/api/v1/scheduled-tasks/{id}` | PUT | 更新任务 |
| `/api/v1/scheduled-tasks/{id}` | DELETE | 删除任务 |
| `/api/v1/scheduled-tasks/{id}/trigger` | POST | 手动触发 |
| `/api/v1/scheduled-tasks/{id}/pause` | POST | 暂停任务 |
| `/api/v1/scheduled-tasks/{id}/resume` | POST | 恢复任务 |
| `/api/v1/scheduled-tasks/{id}/history` | GET | 执行历史 |

---

## 四、任务类型

| 类型 | 说明 |
|------|------|
| 脚本执行 | 执行指定脚本 |
| 巡检任务 | 执行数据库巡检 |
| 定时审批 | 定时执行已审批的变更 |

---

## 五、注意事项

- 服务重启后任务自动恢复
- 任务失败会记录日志
- 可配置失败重试
