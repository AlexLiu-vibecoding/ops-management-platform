# 认证与权限

> JWT Token 认证、RBAC 权限模型、多角色支持

---

## 基本信息

- **规格编号**: `SPEC-001`
- **功能名称**: 认证与权限
- **优先级**: P0 (核心基础)
- **状态**: ✅ 已完成

---

## 一、需求背景

### 1.1 问题描述
系统需要用户认证和权限控制，确保不同角色用户只能访问其权限范围内的功能。

### 1.2 目标
- 用户登录获取 Token
- 基于 Token 的身份验证
- 基于角色的权限控制

### 1.3 不做的事
- OAuth 第三方登录
- 多因素认证
- SSO 单点登录

## 二、数据模型（精确约束）

### 2.1 用户表

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(100) UNIQUE,
    role VARCHAR(20) NOT NULL DEFAULT 'developer',
    is_enabled BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
```

### 2.3 Pydantic Schema

```python
from pydantic import BaseModel, EmailStr
from typing import Optional
from datetime import datetime

class LoginRequest(BaseModel):
    username: str
    password: str

class TokenResponse(BaseModel):
    access_token: str
    refresh_token: str
    token_type: str = "bearer"
    expires_in: int  # 秒

class UserResponse(BaseModel):
    id: int
    username: str
    email: Optional[str]
    role: str
    is_enabled: bool
    created_at: datetime

class UserCreate(BaseModel):
    username: str
    password: str
    email: Optional[EmailStr] = None
    role: str = "developer"

class UserUpdate(BaseModel):
    email: Optional[EmailStr] = None
    role: Optional[str] = None
    is_enabled: Optional[bool] = None
```

## 三、API 接口定义（精确约束）

### 3.1 接口列表

| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/api/v1/auth/login` | POST | 登录 | 无 |
| `/api/v1/auth/logout` | POST | 登出 | 登录用户 |
| `/api/v1/auth/refresh` | POST | 刷新Token | 登录用户 |
| `/api/v1/auth/me` | GET | 当前用户信息 | 登录用户 |
| `/api/v1/users` | GET | 用户列表 | super_admin |
| `/api/v1/users` | POST | 创建用户 | super_admin |
| `/api/v1/users/{id}` | GET | 用户详情 | super_admin |
| `/api/v1/users/{id}` | PUT | 更新用户 | super_admin |
| `/api/v1/users/{id}` | DELETE | 删除用户 | super_admin |

### 3.2 接口详细定义

#### POST /api/v1/auth/login（登录）

**请求体：**
```json
{
  "username": "admin",
  "password": "password123"
}
```

**成功响应（200）：**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "expires_in": 1800
}
```

**错误响应（401）：**
```json
{
  "error": "INVALID_CREDENTIALS",
  "message": "用户名或密码错误"
}
```

**错误响应（403）：**
```json
{
  "error": "ACCOUNT_DISABLED",
  "message": "账户已被禁用"
}
```

#### POST /api/v1/auth/refresh（刷新Token）

**请求头：**
```
Authorization: Bearer {refresh_token}
```

**成功响应（200）：**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "expires_in": 1800
}
```

**错误响应（401）：**
```json
{
  "error": "INVALID_TOKEN",
  "message": "Token无效或已过期"
}
```

#### GET /api/v1/auth/me（当前用户）

**请求头：**
```
Authorization: Bearer {access_token}
```

**成功响应（200）：**
```json
{
  "id": 1,
  "username": "admin",
  "email": "admin@example.com",
  "role": "super_admin",
  "is_enabled": true,
  "created_at": "2024-01-01T00:00:00Z"
}
```

#### GET /api/v1/users（用户列表）

**请求参数：**
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| page_size | int | 否 | 20 | 每页数量，最大100 |
| role | string | 否 | - | 角色筛选 |
| is_enabled | bool | 否 | - | 状态筛选 |

**成功响应（200）：**
```json
{
  "items": [
    {
      "id": 1,
      "username": "admin",
      "email": "admin@example.com",
      "role": "super_admin",
      "is_enabled": true,
      "created_at": "2024-01-01T00:00:00Z"
    }
  ],
  "total": 10,
  "page": 1,
  "page_size": 20
}
```

#### POST /api/v1/users（创建用户）

**请求体：**
```json
{
  "username": "newuser",
  "password": "password123",
  "email": "user@example.com",
  "role": "developer"
}
```

**成功响应（201）：**
```json
{
  "id": 2,
  "username": "newuser",
  "email": "user@example.com",
  "role": "developer",
  "is_enabled": true,
  "created_at": "2024-01-01T00:00:00Z"
}
```

**错误响应（400）：**
```json
{
  "error": "USERNAME_EXISTS",
  "message": "用户名已存在"
}
```

## 四、测试用例（行为约束）

### 4.1 登录测试

```python
def test_login_success(client, test_user):
    """测试登录成功"""
    response = client.post("/api/v1/auth/login", json={
        "username": "testuser",
        "password": "password123"
    })
    assert response.status_code == 200
    data = response.json()
    assert "access_token" in data
    assert "refresh_token" in data
    assert data["token_type"] == "bearer"
    assert data["expires_in"] > 0


def test_login_wrong_password(client, test_user):
    """测试密码错误"""
    response = client.post("/api/v1/auth/login", json={
        "username": "testuser",
        "password": "wrongpassword"
    })
    assert response.status_code == 401
    assert response.json()["error"] == "INVALID_CREDENTIALS"


def test_login_user_not_found(client):
    """测试用户不存在"""
    response = client.post("/api/v1/auth/login", json={
        "username": "notexist",
        "password": "password123"
    })
    assert response.status_code == 401
    assert response.json()["error"] == "INVALID_CREDENTIALS"


def test_login_disabled_user(client, disabled_user):
    """测试禁用用户登录"""
    response = client.post("/api/v1/auth/login", json={
        "username": "disabled",
        "password": "password123"

    assert response.json()["error"] == "ACCOUNT_DISABLED"
```

def test_create_user_success(client, super_admin_token):
