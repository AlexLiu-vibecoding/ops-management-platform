# 通知系统重新设计方案

## 一、当前问题

### 1. 数据库与模型不一致
- 数据库: `start_time`, `end_time`, `recurrence_type`, `recurrence_pattern`
- 模型: `start_date`, `end_date`, `silence_type`, `time_start`, `time_end`, `weekdays`
- 结果: API 返回 500 错误

### 2. 架构问题
- 静默规则和频率限制是全局的，未关联通道
- 只支持钉钉通道（DingTalkChannel）
- 设计不够灵活

---

## 二、新设计方案

### 核心思路
**把静默规则和频率限制关联到通道上**

```
通知通道 (notification_channels)
├── 通道配置
├── 静默规则 (一对多)
└── 频率限制 (一对多)
```

### 数据模型

#### 1. 统一的通知通道表
```sql
CREATE TABLE notification_channels (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    channel_type VARCHAR(20) NOT NULL,  -- dingtalk/wechat/feishu/email/webhook
    
    -- 通道配置 (JSON格式，不同类型配置不同)
    config JSON NOT NULL,
    -- dingtalk: {webhook, auth_type, secret, keywords}
    -- wechat: {webhook, ...}
    -- feishu: {webhook, ...}
    -- email: {smtp_host, smtp_port, username, password, from_addr}
    -- webhook: {url, method, headers, body_template}
    
    -- 默认静默和频率限制
    default_silence_enabled BOOLEAN DEFAULT FALSE,
    default_rate_limit_enabled BOOLEAN DEFAULT FALSE,
    
    is_enabled BOOLEAN DEFAULT TRUE,
    description VARCHAR(200),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. 静默规则表 (关联通道)
```sql
CREATE TABLE channel_silence_rules (
    id SERIAL PRIMARY KEY,
    channel_id INTEGER NOT NULL REFERENCES notification_channels(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    description VARCHAR(200),
    
    -- 匹配条件 (可选)
    alert_level VARCHAR(16),      -- 告警级别
    instance_id INTEGER,          -- 实例ID (通用)
    metric_type VARCHAR(32),      -- 指标类型
    
    -- 静默配置
    silence_type VARCHAR(20) DEFAULT 'once',  -- once/daily/weekly
    start_time TIMESTAMP,         -- 开始时间
    end_time TIMESTAMP,           -- 结束时间
    time_start VARCHAR(5),        -- 每日开始时间 HH:MM
    time_end VARCHAR(5),          -- 每日结束时间 HH:MM
    weekdays INTEGER[],           -- 星期几 [0-6]
    
    is_enabled BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 3. 频率限制表 (关联通道)
```sql
CREATE TABLE channel_rate_limits (
    id SERIAL PRIMARY KEY,
    channel_id INTEGER NOT NULL REFERENCES notification_channels(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    description VARCHAR(200),
    
    -- 匹配条件 (可选)
    alert_level VARCHAR(16),
    instance_id INTEGER,
    metric_type VARCHAR(32),
    
    -- 频率限制
    limit_window INTEGER DEFAULT 300,      -- 时间窗口(秒)
    max_notifications INTEGER DEFAULT 5,   -- 最大通知数
    cooldown_period INTEGER DEFAULT 600,   -- 冷却期(秒)
    
    is_enabled BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 页面结构

```
通知管理
├── 通道管理
│   ├── 通道列表
│   ├── 新建通道 (选择类型: 钉钉/企微/飞书/邮件/Webhook)
│   └── 通道详情
│       ├── 基本信息
│       ├── 静默规则 Tab (该通道的静默规则)
│       └── 频率限制 Tab (该通道的频率限制)
│
├── 通知历史
└── 通知模板
```
