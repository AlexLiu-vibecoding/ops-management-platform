## 一、功能概述

### 1.1 权限模型

RBAC + 数据权限混合模型：

```
用户 → 角色 → 功能权限
       ↓
    环境权限 → 数据权限
```

### 1.2 角色定义

| 角色 | 功能权限 | 数据权限 |
|------|----------|----------|
| super_admin | 全部功能 | 全部环境 |
| approval_admin | 审批+执行 | 授权环境 |
| operator | 执行操作 | 授权环境 |
| developer | 查看+申请 | 授权环境 |

### 1.3 环境权限

环境权限绑定角色，而非用户：

```python
class EnvironmentPermission:
    environment_id: int
    role: UserRole
    permissions: list  # ['view', 'execute', 'admin']
```

## 三、核心接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/permissions/roles` | GET | 角色列表 |
| `/api/v1/permissions/users` | GET | 用户权限列表 |
| `/api/v1/permissions/users/{id}/role` | PUT | 修改用户角色 |
| `/api/v1/permissions/environments/{id}` | GET | 环境权限 |
| `/api/v1/permissions/environments/{id}` | POST | 设置环境权限 |

## 四、权限检查

后端通过依赖注入检查权限：

```python
# 获取当前用户
@router.get("/xxx")
async def xxx(user: User = Depends(get_current_user)):
    pass

# 需要 super_admin
@router.delete("/xxx")
async def xxx(user: User = Depends(get_super_admin)):
    pass

# 需要 operator 及以上
@router.post("/xxx")
async def xxx(user: User = Depends(get_operator)):
    pass
```

## 五、注意事项

- 角色变更立即生效
- 环境权限基于角色而非用户
- developer 角色可查看监控、提交变更申请
