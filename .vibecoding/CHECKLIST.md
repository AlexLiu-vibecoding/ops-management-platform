# 开发检查清单

> AI 在开发过程中的必查项，确保不遗漏关键步骤

---

## 一、开发前检查

```
□ 读取 VIBECODING.md 了解协作经验
□ 读取相关规格文件（specs/xxx.md）
□ 确认需求范围，不擅自扩展
□ 检查现有代码，避免重复造轮子
□ 列出将要修改的文件清单
```

---

## 二、代码修改检查

### 2.1 修改前
```
□ 先 read_file 读取文件内容
□ 了解现有代码结构
□ 确认修改点，不误删现有功能
```

### 2.2 修改时
```
□ 优先使用 edit_file 精准修改
□ 大改动时使用 write_file 覆盖
□ 保持代码风格一致
□ 不修改无关代码
```

### 2.3 修改后
```
□ 确认改动范围与需求一致
□ 没有引入硬编码
□ 没有遗漏 import 语句
```

---

## 三、特定场景检查

### 3.1 新增 API
```
□ 在 main.py 注册路由
□ 添加 Pydantic Schema
□ 添加测试用例（tests/test_xxx_api.py）
□ 前端 API 封装（frontend/src/api/）
```

### 3.2 新增菜单
```
□ 后端 menu.py 添加菜单配置
□ 前端 router/index.js 添加路由
□ 创建页面组件
□ 提示用户调用 /api/v1/menu/add-missing
```

### 3.3 新增数据库字段
```
□ 更新 SQLAlchemy 模型
□ 更新 Pydantic Schema
□ 考虑数据迁移（已有数据的默认值）
□ 更新相关 API
```

### 3.4 新增前端页面
```
□ 创建组件文件
□ 添加路由配置
□ 使用 TableActions 组件（表格操作列）
□ 使用 el-form inline 布局（筛选区域）
```

---

## 四、开发后检查

```
□ 让用户验证效果
□ 检查日志是否有错误
□ 新增测试用例是否通过
□ 更新相关文档
□ 更新 VIBECODING.md（如有新经验）
```

---

## 五、验收确认模板

开发完成后，按以下格式向用户确认：

```
## 完成内容

1. 修改文件：
   - frontend/src/xxx.vue - 改动说明
   - backend/app/api/xxx.py - 改动说明

2. 验收要点：
   - [ ] 功能点1
   - [ ] 功能点2

3. 请验证：
   - 刷新页面查看 xxx 功能
   - 测试 xxx 场景
```

---

*使用方式：开发过程中对照检查，每完成一项打勾确认*
